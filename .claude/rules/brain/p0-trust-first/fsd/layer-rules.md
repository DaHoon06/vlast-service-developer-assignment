# 기본 원칙 · 경계 기준

## 레이어 단방향 의존성

- **순서**: `app` → `pages` → `widgets` → `features` → `entities` → `shared`
- 하위 layer는 상위 레이어를 **import하지 않는다**.
    - **같은 layer** slice 간 import는 **금지**

## shared 레이어

- **segment가 경계** (slice 없음).
- 각 segment 루트에 public API (`index.ts`) 구성.
- 직접 깊은 경로 접근은 가능하나, **segment 루트 경유를 권장** (선택).

## pages / widgets / features / entities

- **slice가 경계이며 하위 segment 존재**.
- slice 루트에 `index.ts` **필수**.
    - 외부에서는 **slice 루트만** import.

### 세그먼트 역할

슬라이스 **내부** segment다. slice 경계와 혼동하지 않는다

#### `ui`

- **역할**: UI 표시·표현.
- **포함**: 컴포넌트, 포맷터, 스타일 등.

#### `api`

- **역할**: 백엔드와의 통신.
- **포함**: 요청 함수, (쿼리 옵션 등) 데이터 타입, React Query 설정, 매퍼.

#### `model`

- **역할**: 데이터 구조·상태·도메인 규칙.
- **포함**: 스키마, 인터페이스, 스토어, 비즈니스 로직.

#### `lib`

- **역할**: **해당 슬라이스 내부** 보조 코드.
- **포함**: 슬라이스 전용 헬퍼·유틸.
- **주의**: 여러 슬라이스에서 쓰이면 `shared/lib`로 이동.

#### `config`

- **역할**: 슬라이스별 설정.
- **포함**: 환경별 설정, feature flag, 상수.

## 레이어 배치 기준

### entities

- **도메인 객체**가 기준. 비즈니스에서 독립적으로 식별되는 데이터 단위(Member, Character, Ranking 등).
- 역할: **표시(표현)만**. 자체 인터랙션을 갖지 않는다.
- 포함: 도메인 타입, 조회 API, 표시 전용 UI 컴포넌트.

### features

- **단일 사용자 인터랙션 목표**가 기준. 사용자가 달성하려는 하나의 액션.
- 역할: 인터랙션 처리. 상태 변경, API 호출, 폼 제출 등.
- 포함: 인터랙션 UI, 해당 액션에 필요한 API, 로컬 상태.
- 판단 질문: "이것은 사용자가 수행하는 하나의 행동인가?" (예: 닉네임 입력, 캐릭터 선택, 랭킹 열람)

### widgets

- **features + entities 조합**이 기준. 도메인 단위가 아닌 UI 조합 블록.
- 역할: 페이지에 직접 삽입되는 자급자족 UI 블록. 여러 features/entities를 조합해 독립적으로 동작.
- 포함: 조합 로직, 해당 블록 전용 하위 UI, 블록 수준의 상태·훅.
- 판단: "이것이 도메인 개념이 아니라 여러 기능을 묶은 UI 덩어리인가?" (예: BakingBoard — 게임 로직 전체를 담당하는 독립 블록)

### 판단 플로우

```
새 슬라이스를 어디에 둘까?
├─ 비즈니스 도메인 객체(데이터 모델 중심)? ─→ entities
├─ 사용자 인터랙션 하나? ─────────────────→ features
├─ 여러 features/entities 조합 블록? ─────→ widgets
└─ 페이지 진입점(라우트 조합)? ────────────→ pages
```

## @x cross-import

### 개요

- 같은 레이어 슬라이스 간 import는 원칙적으로 금지된다.
- **`@x`**는 그 제약을 우회하는 **FSD 공식 cross-import 패턴**(v2.1 표준).
- `A/@x/B`는 **"A crossed with B"**로 읽는다.
- 다른 슬라이스에 **노출할 타입·데이터만** `@x`에 명시적으로 둔다.

### 파일 규칙

- 경로: `{slice}/@x/{consumer-slice}.ts`
- 각 파일은 **특정 소비자 슬라이스 전용** export만 포함.
- 파일명 = 소비자 슬라이스 이름 → 의존 방향이 드러난다.
- 기존 slice 루트 `index.ts`(public API)는 그대로 유지.

### 사용 기준

- **entities 레이어에서만** 사용한다 (엔티티 간 참조가 불가피할 때).
- 데이터 fetch 함수·비즈니스 로직 공유 용도로 쓰지 않는다 → **상위 레이어에서 조합**.
- `@x`가 과하면 슬라이스 병합 또는 상위 레이어 추출을 검토한다.
